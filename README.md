Metro Manila Traffic Incident Analysis
An End-to-End Data Pipeline: Python → SQL → Power BI

📌 Executive Summary
This project analyzes 17.3K traffic incidents in Metro Manila to identify high-risk patterns for commuters and city planners. By processing raw data through a full technical stack, I identified that 7:00 AM in Quezon City represents the single highest risk window for vehicular accidents.

🛠️ Technical Stack
Data Cleaning: Python (Pandas) for handling missing values and data type standardization.

Database Management: SQL for complex aggregations and filtering of incident categories.

Visualization: Power BI for interactive dashboarding and trend analysis.

📊 Key Insights
Peak Risk Window: Incident volume peaks sharply at 7 AM (1,354 recorded incidents), directly aligning with the morning rush hour surge.

Geographic Hotspot: Quezon City (8.7K incidents) reports nearly triple the volume of the next highest city, marking it as a critical area for traffic intervention.

Incident Breakdown: Vehicular Accidents (82.6%) are the primary driver of congestion, followed by mechanical stalls.
